# 🎅 Проект «Тайный Санта» — Telegram-бот
https://t.me/WishSecretSantaBot

## 1. Описание проекта

Проект представляет собой Telegram-бота для организации игры «Тайный Санта» в группах и чатах друзей. Бот автоматизирует все этапы игры: создание игры, регистрацию участников, сбор пожеланий, проведение жеребьёвки, анонимное уведомление каждого участника о том, кому он дарит подарок, а также предоставляет дополнительные функции, такие как установка бюджета и дедлайна, напоминания и анонимные вопросы своему Санте.

Проект выполнен в рамках учебного задания и включает все обязательные компоненты пройденного материала: условия, словари, функции, работа с файлами. Также добавлена оригинальная идея — возможность анонимно задать вопрос своему Тайному Санте.

## 2. Функциональные возможности

### 2.1. Основной сценарий игры

- **Создание игры** – любой участник группы (становится администратором) отправляет команду `/new_game`. Бот создаёт игровую сессию, связанную с данным чатом.
- **Регистрация участников** – каждый желающий пишет боту в личные сообщения команду `/join`. Бот проверяет наличие активных игр и добавляет пользователя в список участников. Если игр несколько, пользователь выбирает нужную с помощью инлайн-кнопок.
- **Указание пожеланий** – участник отправляет боту в личку `/set_wish` с текстом пожелания. Можно сразу указать имя, под которым участник хочет фигурировать в игре (формат: `Имя: пожелание`). Если имя не указано, бот использует username или first name из Telegram.
- **Жеребьёвка** – администратор в группе запускает жеребьёвку командой `/start_draw`. Бот проверяет, что участников не меньше двух и все указали пожелания, затем случайным образом распределяет пары (гарантируя, что никто не дарит сам себе).
- **Уведомления** – каждому участнику в личные сообщения приходит сообщение вида:  
  `🎅 Ты даришь подарок пользователю Анна. Его пожелание: хочу книгу`.
- **Статус игры** – команда `/status` в группе показывает текущее состояние: список участников с отметками о пожеланиях, бюджет, дедлайн.

### 2.2. Дополнительные возможности

- **Установка бюджета** – администратор может задать бюджет командой `/set_budget` (например, `/set_budget 1000 руб.`).
- **Установка дедлайна** – администратор может установить дату окончания игры командой `/set_deadline` (например, `/set_deadline 2025-12-31`).
- **Напоминания** – команда `/remind` отправляет личные напоминания участникам, которые ещё не указали пожелания.
- **Анонимные вопросы Санте (оригинальная идея)** – после жеребьёвки получатель подарка может анонимно задать вопрос своему Санте с помощью команды `/ask_question Текст вопроса` в личных сообщениях боту. Вопрос пересылается Санте без указания отправителя.

## 3. Техническая реализация


### 3.1. Структура проекта

```
bot.py               # Основной файл с кодом бота
santa_data.json      # Файл с данными игр и пользователей (создаётся автоматически)
config.py            # Токен
README.md            # Описание проекта (данный файл)
```

### 3.2. Ключевые компоненты кода (в соответствии с требованиями)

#### 3.2.1. Условия
В коде широко используются условные операторы для проверки:
- прав администратора (`is_admin`),
- статуса игры (регистрация, жеребьёвка проведена),
- наличия участников и их пожеланий,
- типа чата (группа или личные сообщения),
- корректности ввода команд.

Пример:
```python
if chat.type == "private":
    await update.message.reply_text("❌ Эту команду нужно использовать в группе.")
    return
if not is_admin(user.id, chat_id, data):
    await update.message.reply_text("❌ Только администратор может запустить жеребьёвку.")
    return
```

#### 3.2.2. Словари
Все данные организованы в виде словарей:
- `games` – словарь игр, где ключ — ID чата, значение — словарь с параметрами игры (админ, участники, пожелания, назначения и т.д.).
- `user_chats` – словарь, сопоставляющий user_id с chat_id для отправки личных сообщений.
- `users` – словарь с информацией о пользователях (username, first_name, last_name, display_name — имя, указанное вручную).

Пример:
```python
data["games"][chat_id] = {
    "admin": str(user.id),
    "participants": [],
    "wishes": {},
    "assignments": {},
    "status": "registration",
    "budget": None,
    "deadline": None
}
```

#### 3.2.3. Функции
Код разбит на множество функций, каждая из которых отвечает за отдельное действие:
- `load_data()`, `save_data()` – загрузка и сохранение данных.
- `is_admin()`, `get_user_display_name()`, `update_user_info()`, `parse_wish_input()` – вспомогательные функции.
- Асинхронные обработчики команд: `start`, `new_game`, `join`, `set_wish`, `start_draw`, `status`, `set_budget`, `set_deadline`, `remind`, `ask_question`.
- Обработчики callback-запросов от инлайн-кнопок: `join_callback`, `wish_callback`, `ask_callback`.

#### 3.2.4. Чтение и создание файлов
При запуске бот загружает данные из файла `santa_data.json` (функция `load_data`). Если файл отсутствует или пуст, создаётся словарь с начальной структурой. При каждом изменении данных (регистрация, добавление пожелания, жеребьёвка) вызывается `save_data`, которая записывает актуальное состояние в тот же файл в формате JSON с отступами для читаемости.

Пример:
```python
def save_data(data):
    with open(DATA_FILE, "w", encoding="utf-8") as f:
        json.dump(data, f, ensure_ascii=False, indent=2)
```

### 3.3. Оригинальная идея

В качестве оригинальной идеи реализована функция **анонимных вопросов своему Санте**. После проведения жеребьёвки любой участник, который является получателем подарка (то есть ему дарят), может отправить боту команду `/ask_question` с текстом вопроса. Бот определяет, кто является Сантой для этого пользователя (то есть кто ему дарит), и пересылает вопрос в личные сообщения Санте **без указания отправителя**. Это позволяет сохранить интригу и при необходимости уточнить детали подарка (цвет, размер, предпочтения) анонимно.

Данная функция расширяет стандартную логику «Тайного Санты» и делает игру более интерактивной и удобной.
